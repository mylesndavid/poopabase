# The database for agents

Agents don't fit neatly into one database paradigm. They need SQL for structured data, document search for knowledge, vector search for semantics, NoSQL for flexibility, and memory to remember things across conversations.

poopabase gives you all five in a single file.

```
┌─────────────────────────────────────────────┐
│                 poopabase                    │
│                                              │
│  ┌─────────┐ ┌────────────┐ ┌────────────┐  │
│  │   SQL   │ │ Collections│ │ Documents  │  │
│  │  tables │ │   (NoSQL)  │ │   (FTS5)   │  │
│  └─────────┘ └────────────┘ └────────────┘  │
│  ┌─────────────────┐ ┌──────────────────┐   │
│  │  Hybrid Search  │ │  Agent Memory    │   │
│  │ BM25 + Vectors  │ │ observe → recall │   │
│  └─────────────────┘ └──────────────────┘   │
│                                              │
│            ┌──────────────┐                  │
│            │  libSQL/SQLite│                  │
│            │  .poop.db    │                  │
│            └──────────────┘                  │
└─────────────────────────────────────────────┘
         ▲         ▲         ▲
         │         │         │
      CLI (poop)  HTTP API  MCP Server
```

## Why poopabase exists

Most agents today cobble together 3-4 different services: a database, a vector store, a document index, and some kind of memory layer. That's a lot of infra for a thing that just needs to remember what you told it.

poopabase collapses all of that into a single `.poop.db` file powered by [libSQL](https://github.com/tursodatabase/libsql) (the SQLite fork from Turso). No servers to manage. No docker compose files with six services. One file, one process.

**It works three ways:**

- **`poop` CLI** — manage everything from your terminal
- **HTTP API** — `poop serve` starts a REST server on port 3141
- **MCP Server** — plug directly into Claude Code, Cursor, Windsurf, or any MCP-compatible agent

## Built on libSQL

Under the hood, poopabase is libSQL — which means it's SQLite with extras:

- **FTS5** for full-text search with BM25 ranking
- **F32_BLOB** for native vector storage and cosine similarity
- **JSON operators** (`->>`) for querying nested documents
- **WAL mode** for concurrent reads
- Everything lives in a single file you can `cp`, `scp`, or commit to git

<Note>
  poopabase is not a hosted service. It's a local-first database that runs wherever your agent runs. Think SQLite, not Postgres.
</Note>
